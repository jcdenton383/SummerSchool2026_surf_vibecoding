# SCR-012. Добавление участников

## Метаданные

| Поле | Значение |
|---|---|
| Раздел приложения | Аккаунт / Бронирование |
| Приоритет | Высокий |
| Что реализует | Добавление новых участников в существующую предстоящую запись |
| Зависит от экранов | SCR-011 Детали брони |
| Следующие экраны | SCR-007 Оплата и подтверждение; SCR-011 Детали брони |

## Что пользователь должен понять и сделать

Пользователь должен понять, что он добавляет только новых участников к уже существующей записи, а не редактирует весь состав. Основное действие - заполнить данные новых участников и перейти к подтверждению доплаты.

## Какой контент показать и что в нём главное

Главное - текущий состав, доступные места и новые участники.

Показать:
- краткую сводку брони;
- текущих участников без возможности перепутать их с новыми;
- количество оставшихся мест;
- карточки новых участников;
- имя, аллергии и инвентарь для каждого нового участника;
- предварительную доплату;
- кнопку «Продолжить».

## Какие состояния нарисовать

- Можно добавить одного участника.
- Можно добавить нескольких участников.
- Новых участников больше, чем свободных мест.
- Не заполнены обязательные данные.
- Выбран прокат или собственный инвентарь.
- Возврат к деталям без изменений.
