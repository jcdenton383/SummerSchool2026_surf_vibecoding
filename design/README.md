# Постановки на дизайн экранов

Папка содержит постановки для UX/UI-дизайнера по экранам из `screen_registry.md`.

Каждый файл отвечает на вопросы:
- что пользователь должен понять и сделать;
- какой контент показать и что в нём главное;
- какие состояния нужно нарисовать;
- к какому разделу приложения относится экран;
- какой приоритет у экрана;
- что экран реализует;
- от каких экранов зависит.

Постановки намеренно не описывают API, коды ответов и техническую реализацию.

## Системные дизайн-артефакты

- `ER-model-and-entity-models.md` - логическая ER-модель, описание сущностей, режимы чтения/изменения данных и sequence-диаграмма клиентского сценария с ветками.
- `entity-access-matrix.md` - роли, матрица доступа к сущностям и правила операций клиентского приложения.
- `SEQ-booking-creation.md` - sequence-диаграмма создания брони с ветками `201`, `409`, `410`.
