# Версия 1.1 — глобальная навигация в верхней панели

## Тип

Feature / UX refinement.

## Затронутые экраны

- Все экраны клиентского приложения.
- Базовые целевые экраны: `SCR-003` и `SCR-010`.

## Цель

Пользователь должен иметь постоянный доступ к расписанию и своим записям без зависимости только от кнопки «Назад» и контекстных кнопок внутри экранов.

## UI-решение

В правой части верхней панели, напротив системной кнопки «Назад», размещаются две компактные placeholder-кнопки:

- `Расписание`;
- `Мои записи`.

Кнопки находятся на той же панели, где отображаются заголовки экранов.

## Поведение

| Кнопка | Условие | Результат |
|---|---|---|
| `Расписание` | любое состояние | переход на `SCR-003` |
| `Мои записи` | пользователь авторизован | переход на `SCR-010` |
| `Мои записи` | пользователь не авторизован | переход на `SCR-001`, после входа возврат на `SCR-010` |

## Поведение при незавершённой заявке

Если пользователь уходит через глобальные кнопки из процесса оформления:

- на этапе `SCR-006` данные формы не сохраняются;
- на этапе `SCR-007` и `SCR-008` черновик заявки очищается, бронь не создаётся;
- если бронь уже создана на `SCR-009`, она не отменяется автоматически.

## Ограничения

- Глобальные кнопки не должны сбрасывать авторизацию клиента.
- Глобальные кнопки не должны создавать бронь.
- Онлайн-оплата не добавляется.
