# Версия 1.1 — ревизия матрицы доступа к сущностям

## Статус

`revision`

## Основание

Документ фиксирует изменения версии 1.1 к `design/entity-access-matrix.md`.

## Изменения доступа

| Сущность | Клиентское приложение | Клиент | Backend | Админка/внешняя система | Комментарий |
|---|---|---|---|---|---|
| `ClassProgram.dishImageUrl` | `R` | `R` | `R` | `C/R/U` вне приложения | Non-MVP 1.1. Приложение только отображает фото блюда. |
| `Booking` активная уникальность по `clientId + slotId` | `R` | `R` | `C/R/U` с проверкой ограничения | не требуется в MVP | Backend запрещает вторую активную бронь на тот же слот для клиента. |

## Решение

1. Клиентское приложение не получает права создавать или редактировать фото блюд.
2. Backend остаётся источником истины для изображения блюда и активных броней.
3. При уже существующей активной брони клиентское приложение должно вести пользователя в сценарий добавления участников, а не в создание новой брони.

## Non-MVP отметка

Фото блюда является расширением версии 1.1 и не входит в MVP. Базовое приложение должно оставаться работоспособным при отсутствии `dishImageUrl`.
