# SCR-006. Оформление записи: участники

## Метаданные

| Поле | Значение |
|---|---|
| Раздел приложения | Бронирование |
| Приоритет | Высокий |
| Что реализует | Одиночную и групповую запись с данными участников, аллергиями и выбором инвентаря |
| Зависит от экранов | SCR-005 Карточка класса |
| Следующие экраны | SCR-007 Оплата и подтверждение; SCR-016 Условия и политика данных |

## Что пользователь должен понять и сделать

Пользователь должен понять, кого он записывает на класс, какие данные обязательны, и заполнить информацию по каждому участнику. Главное действие - корректно заполнить участников и перейти к подтверждению.

## Какой контент показать и что в нём главное

Главное - участники и обязательные поля по каждому из них. Нужно явно показать, что аллергии выбираются для каждого участника отдельно.

Показать:
- краткую сводку выбранного класса;
- количество доступных мест;
- данные основного клиента;
- карточки участников;
- имя каждого участника;
- переключатель «аллергии есть / нет»;
- поле комментария при наличии аллергии;
- выбор «свой инвентарь / прокат»;
- действие добавления участника;
- предварительную сумму или подсказку, что итог будет на следующем шаге;
- кнопку «Продолжить».

## Какие состояния нарисовать

- Один участник.
- Несколько участников.
- Добавление нового участника.
- Не выбран статус аллергии.
- Выбраны аллергии, но не заполнен комментарий.
- Выбран прокат инвентаря.
- Выбран собственный инвентарь.
- Свободных мест меньше, чем участников.
- Загрузка при переходе к подтверждению.
