# SCR-002. Код подтверждения

## Метаданные

| Поле | Значение |
|---|---|
| Раздел приложения | Авторизация |
| Приоритет | Высокий |
| Что реализует | Подтверждение телефона и завершение входа |
| Зависит от экранов | SCR-001 Ввод телефона |
| Следующие экраны | SCR-003 Расписание классов; SCR-010 Аккаунт / мои записи |

## Что пользователь должен понять и сделать

Пользователь должен понять, что код отправлен на указанный номер, и ввести его для входа. Если номер указан неверно, пользователь должен иметь простой способ вернуться и изменить его.

## Какой контент показать и что в нём главное

Главное - поле кода и подтверждение действия. Номер телефона нужен как ориентир, но не должен перетягивать внимание.

Показать:
- заголовок подтверждения;
- замаскированный или полный номер телефона;
- поле ввода кода;
- таймер или состояние повторной отправки;
- кнопку «Войти»;
- действие «Изменить номер».

## Какие состояния нарисовать

- Ожидание ввода кода.
- Код введён не полностью.
- Неверный код.
- Истёкший код.
- Повторная отправка кода доступна.
- Проблема соединения.
- Загрузка при подтверждении.
