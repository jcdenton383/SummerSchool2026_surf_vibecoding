# Таблица ручного тестирования MVP

## Назначение

Артефакт предназначен для ручной приемочной и регрессионной проверки учебного MVP мобильного приложения "Шеф-стол". Таблица опирается на известные требования, OpenAPI-контракт, реализованный frontend, mock-backend и уже заведенные баг-логи.

## Правила ручного прогона

- P0 тесты выполнять на Android и iOS. Проверка только на Expo Web не засчитывается как мобильная приемка.
- Для каждого теста фиксировать фактический результат, статус и ссылку на баг-лог, если найден дефект.
- Если тест не может быть выполнен из-за окружения, ставить `blocked`, а не `passed`.
- Production-оплата, админка, CRM, интерфейс шефа и реальные push/SMS не проверяются: это вне MVP.
- Перед прогоном желательно выполнить быстрые технические проверки: frontend typecheck/test, backend health, доступность mock-backend и БД.

## Статусы

| Статус | Значение |
|---|---|
| `not_run` | Тест еще не выполнялся. |
| `passed` | Фактический результат совпал с ожидаемым. |
| `failed` | Найден дефект или расхождение с MVP. |
| `blocked` | Проверка невозможна из-за окружения, данных или зависимостей. |
| `skipped` | Тест сознательно пропущен с указанием причины. |
