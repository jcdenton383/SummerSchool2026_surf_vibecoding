# Спецификация экранов

Документ использует структуру экранного ТЗ из шаблона: назначение, навигация, данные, API, элементы, состояния, действия и критерии приёмки.

## Общие правила экранов

| Правило | Требование |
|---|---|
| Авторизация | Просмотр расписания и карточки класса доступен гостю; создание, изменение и отмена брони требуют авторизации. |
| Источник данных | Все бизнес-состояния, места, статусы и брони отображаются по API. |
| Загрузка | Для списков использовать skeleton/loader; для критичных кнопок - loader на кнопке и блокировку повторного тапа. |
| Ошибки | Ошибки сети/API показывать понятным текстом и действием повтора. |
| Empty state | Для пустого расписания использовать текст "Пока нет доступных классов". |
| Персональные данные | Не показывать данные чужих броней; аллергии отображать только в контексте своей брони. |

## SCR-001. Ввод телефона

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Авторизация |
| Приоритет | Высокий |
| API | `POST /auth/phone-code`, `GET /legal/terms` |

Назначение: начать вход или регистрацию клиента по номеру телефона.

Элементы:
- название приложения/студии;
- поле телефона с маской `+7`;
- кнопка "Получить код";
- ссылка на условия и политику данных;
- текст о входе по SMS-коду.

Валидация:
- телефон обязателен;
- формат E.164, пример `+79991234567`;
- при неверном формате показывать ошибку у поля.

Состояния:
- пустое поле;
- неверный формат;
- отправка кода;
- rate limit;
- ошибка сети;
- успешная отправка.

Действия:
- "Получить код" -> `requestPhoneCode` -> SCR-002;
- ссылка условий -> SCR-016.

Критерии приёмки:
- пользователь не может отправить пустой или некорректный телефон;
- при `202` открывается экран кода;
- при `429` показывается таймер/текст ожидания;
- при ошибке сети можно повторить запрос.

## SCR-002. Код подтверждения

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Авторизация |
| Приоритет | Высокий |
| API | `POST /auth/verify-code`, `POST /auth/phone-code` |

Назначение: подтвердить телефон и получить клиентскую сессию.

Элементы:
- заголовок подтверждения;
- замаскированный телефон;
- поле кода;
- таймер повторной отправки;
- кнопка "Войти";
- действие "Изменить номер".

Состояния:
- ввод кода;
- неверный код;
- истёкший код;
- повторная отправка;
- успешный вход;
- ошибка сети.

Действия:
- "Войти" -> `verifyPhoneCode` -> SCR-003 или исходный защищённый экран;
- "Отправить код повторно" -> `requestPhoneCode`;
- "Изменить номер" -> SCR-001.

Критерии приёмки:
- при `200` сохраняется access token;
- при `401` показывается ошибка кода;
- повторная отправка доступна только после таймера;
- новый клиент создаётся бэкендом без отдельной формы регистрации.

## SCR-003. Расписание классов

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Расписание |
| Приоритет | Высокий |
| API | `GET /slots` |

Назначение: показать доступные классы и дать выбрать подходящий слот.

Элементы:
- баннер студии со статическим UI/UX-текстом названия и адреса;
- список классов;
- карточка класса: название, дата, время, длительность, направление, сложность, шеф, рейтинг, свободные места, цена;
- действие выбора даты;
- переход в аккаунт.

Состояния:
- загрузка;
- список классов;
- пустой список "Пока нет доступных классов";
- ошибка сети;
- слот без мест;
- слот отменён студией.

Действия:
- открыть фильтр даты -> SCR-004;
- тап по доступному классу -> SCR-005;
- тап по аккаунту -> SCR-010 или авторизация.

API:
- без параметров `GET /slots` возвращает ближайшие 7 дней;
- с параметрами `from`, `to` возвращает выбранный период.

Критерии приёмки:
- по умолчанию отображается ближайший период;
- поиск и маркетинговые фильтры отсутствуют;
- недоступные слоты не ведут к бронированию;
- адрес студии, если показан, является статическим макетным текстом и не загружается через API.

## SCR-004. Фильтр даты

| Поле | Значение |
|---|---|
| Тип | Экран или bottom sheet |
| Раздел | Расписание |
| Приоритет | Высокий |
| API | `GET /slots?from={date}&to={date}` |

Назначение: выбрать период расписания.

Элементы:
- выбор даты начала;
- выбор даты окончания;
- быстрый сброс к ближайшим 7 дням;
- кнопка "Применить";
- кнопка закрытия.

Валидация:
- дата начала не позже даты окончания;
- диапазон не должен быть пустым;
- некорректный диапазон не отправляется.

Состояния:
- выбран период;
- ошибка периода;
- применение;
- сброс.

Критерии приёмки:
- после применения SCR-003 обновляет список;
- сброс возвращает дефолтный диапазон;
- дополнительные фильтры не появляются.

## SCR-005. Карточка класса

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Расписание / бронирование |
| Приоритет | Высокий |
| API | `GET /slots/{slotId}` |

Назначение: дать полную информацию о классе перед записью.

Элементы:
- название класса;
- дата, время, длительность;
- направление и сложность;
- меню/блюда;
- шеф и рейтинг;
- свободные места и вместимость;
- стоимость места;
- требования к инвентарю;
- описание проката и цена проката;
- правило отмены за 24 часа;
- кнопка "Записаться".

Состояния:
- загрузка;
- доступный слот;
- нет свободных мест;
- слот отменён студией;
- ошибка загрузки.

Действия:
- "Записаться" -> авторизация при необходимости -> SCR-006;
- "Условия" -> SCR-016;
- назад -> SCR-003.

Критерии приёмки:
- бронирование недоступно при `full` и `cancelled_by_studio`;
- цена и свободные места берутся из API;
- рейтинг шефа отображается, если пришёл в данных.

## SCR-006. Оформление записи: участники

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Бронирование |
| Приоритет | Высокий |
| API | локальная форма до SCR-007; актуальность слота при необходимости через `GET /slots/{slotId}` |

Назначение: собрать данные основного клиента и участников.

Элементы:
- сводка класса;
- доступные места;
- имя основного клиента;
- телефон основного клиента из сессии;
- карточки участников;
- имя участника;
- переключатель аллергии есть/нет;
- комментарий при наличии аллергии;
- выбор собственный инвентарь/прокат;
- цена проката;
- добавление участника;
- предварительная сумма;
- кнопка "Продолжить".

Валидация:
- имя основного клиента обязательно;
- имя каждого участника обязательно;
- статус аллергии обязателен по каждому участнику;
- при `has_allergy` комментарий обязателен;
- число участников не больше `availableSeats`;
- выбор инвентаря обязателен по каждому участнику.

Состояния:
- один участник;
- несколько участников;
- добавление участника;
- не выбран статус аллергии;
- выбранная аллергия без комментария;
- выбран прокат;
- выбран собственный инвентарь;
- мест меньше, чем участников;
- переход к подтверждению.

Действия:
- "Добавить участника" добавляет локальную карточку, если есть места;
- "Удалить участника" удаляет локального дополнительного участника;
- "Продолжить" -> SCR-007 при валидной форме.

Критерии приёмки:
- без статуса аллергии продолжить нельзя;
- аллергии не сохраняются в профиль;
- сумма учитывает места и прокат;
- при превышении мест показывается ошибка.

## SCR-007. Оплата и подтверждение

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Бронирование |
| Приоритет | Высокий |
| API | `POST /bookings`, `POST /bookings/{bookingId}/participants` |

Назначение: показать итог записи и отправить подтверждение в API.

Элементы:
- сводка класса;
- список участников;
- выбранный инвентарь;
- стоимость мест;
- стоимость проката;
- итоговая сумма;
- выбор оплаты: оффлайн, онлайн-заглушка;
- правило отмены за 24 часа;
- ссылки на условия и политику;
- кнопка "Подтвердить запись".

Состояния:
- оффлайн-оплата выбрана;
- онлайн-оплата выбрана;
- отправка запроса;
- успешное создание;
- нет мест/устаревшие данные;
- слот отменён;
- ошибка API/сети.

Действия:
- "Подтвердить запись" при оффлайн -> `createBooking` или `addBookingParticipants`;
- выбор онлайн -> SCR-008;
- условия -> SCR-016.

Критерии приёмки:
- бронь не создаётся локально до `201`/`200`;
- при `409`/`410` показывается понятная ошибка и обновляются данные слота;
- при `503` можно повторить;
- при успехе переход на SCR-009 или SCR-011 для добавления участников.

## SCR-008. Онлайн-оплата недоступна

| Поле | Значение |
|---|---|
| Тип | Экран или modal |
| Раздел | Бронирование / оплата |
| Приоритет | Средний |
| API | нет собственного запроса; после согласия используется запрос SCR-007 |

Назначение: объяснить, что онлайн-оплата в MVP недоступна, и предложить оффлайн-оплату.

Элементы:
- сообщение о недоступности онлайн-оплаты;
- предложение оплатить перед классом;
- кнопка "Выбрать оффлайн-оплату";
- кнопка отказа/возврата.

Критерии приёмки:
- при согласии выбран `offline` и пользователь возвращается к подтверждению или сразу отправляет бронь;
- при отказе бронь не создаётся;
- текст не обещает реальную онлайн-оплату.

## SCR-009. Успешная запись

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Бронирование |
| Приоритет | Высокий |
| API | данные из ответа `POST /bookings`; опционально `GET /bookings/{bookingId}` |

Назначение: подтвердить создание брони.

Элементы:
- статус успеха;
- название класса;
- дата и время;
- статический UI/UX-текст адреса студии, если предусмотрен макетом;
- участники;
- способ оплаты;
- итоговая сумма;
- напоминание об инвентаре, если выбран свой;
- правило отмены;
- кнопка "Детали записи";
- кнопка "К расписанию".

Критерии приёмки:
- отображаются данные созданной брони;
- бронь доступна из аккаунта;
- при выборе своего инвентаря есть напоминание.

## SCR-010. Аккаунт / мои записи

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Аккаунт |
| Приоритет | Высокий |
| API | `GET /me`, `GET /bookings/my` |

Назначение: показать клиенту его записи.

Элементы:
- минимальные данные клиента;
- список предстоящих записей;
- список завершённых записей;
- статусы броней;
- метка отмены студией;
- действие оценки для завершённых без отзыва;
- переход к расписанию.

Состояния:
- загрузка;
- нет записей;
- ошибка загрузки;
- предстоящие;
- завершённые;
- отменённые.

Действия:
- открыть бронь -> SCR-011;
- оценить шефа -> SCR-015;
- расписание -> SCR-003.

Критерии приёмки:
- отображаются только свои брони;
- статусы едины с общим словарём;
- отменённые студией видны явно.

## SCR-011. Детали брони

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Аккаунт / бронирование |
| Приоритет | Высокий |
| API | `GET /bookings/{bookingId}` |

Назначение: показать полную информацию по существующей брони и доступные действия.

Элементы:
- статус брони;
- класс, дата, время, адрес;
- участники и их статусы;
- выбранный инвентарь;
- способ оплаты и сумма;
- правило отмены;
- кнопка "Добавить участников";
- кнопка "Отменить";
- кнопка "Оценить шефа" для завершённой брони.

Состояния:
- активная предстоящая бронь;
- менее 24 часов до класса;
- отменена клиентом;
- отменена студией;
- завершена;
- ошибка загрузки.

Критерии приёмки:
- действия отмены и добавления скрыты/заблокированы, если недоступны;
- при отмене студией показывается инструкция по возврату;
- данные аллергий показываются только в рамках этой брони.

## SCR-012. Добавление участников

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Бронирование |
| Приоритет | Высокий |
| API | `POST /bookings/{bookingId}/participants` |

Назначение: добавить новых участников в существующую предстоящую бронь.

Особенности:
- экран повторяет паттерн SCR-006;
- показывает текущий состав брони;
- редактируются только новые участники;
- отображаются оставшиеся свободные места;
- итогом является доплата/новая сумма.

Критерии приёмки:
- нельзя добавить больше участников, чем свободных мест;
- изменение считается успешным только после ответа API;
- при конфликте мест данные обновляются.

## SCR-013. Отмена записи

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Бронирование |
| Приоритет | Высокий |
| API | `POST /bookings/{bookingId}/cancel`, `POST /bookings/{bookingId}/participants/{participantId}/cancel` |

Назначение: подтвердить отмену всей брони или отдельных участников.

Элементы:
- выбор "вся запись" или отдельные участники;
- список участников;
- предупреждение о последствиях;
- заглушка возврата денег;
- итоговое подтверждение;
- кнопка отмены действия.

Состояния:
- отмена разрешена;
- менее 24 часов до класса;
- отправка;
- успех;
- отказ API.

Критерии приёмки:
- менее чем за 24 часа действие заблокировано;
- после успеха пользователь возвращается к SCR-011 с обновлённым статусом;
- удаление не используется, только статусы отмены.

## SCR-014. Отмена класса студией

| Поле | Значение |
|---|---|
| Тип | Экран или состояние SCR-011 |
| Раздел | Аккаунт / уведомления |
| Приоритет | Высокий |
| API | `GET /bookings/{bookingId}`, `GET /notifications/my` |

Назначение: сообщить клиенту об отмене класса студией.

Элементы:
- заметный статус "Класс отменён студией";
- класс, дата, время, адрес;
- причина, если пришла;
- инструкция по возврату;
- пояснение, что повторная запись на слот невозможна;
- переход к расписанию.

Критерии приёмки:
- статус виден из аккаунта и деталей брони;
- push/SMS приводят к актуальному состоянию;
- запись на отменённый слот заблокирована.

## SCR-015. Оценка шефа

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Аккаунт / отзывы |
| Приоритет | Средний |
| API | `POST /chef-reviews` |

Назначение: собрать обратную связь после завершённого класса.

Элементы:
- название класса;
- имя шефа;
- выбор 1-5 звёзд;
- необязательное поле отзыва;
- кнопка "Отправить".

Валидация:
- оценка обязательна;
- текст отзыва необязателен.

Состояния:
- оценка не выбрана;
- отправка;
- успех;
- уже отправлена;
- ошибка API.

Критерии приёмки:
- оценка доступна только для `completed`;
- повторная отправка обрабатывается как "оценка уже отправлена";
- после успеха пользователь возвращается в аккаунт.

## SCR-016. Условия и политика данных

| Поле | Значение |
|---|---|
| Тип | Экран |
| Раздел | Правовая информация |
| Приоритет | Высокий |
| API | `GET /legal/terms` |

Назначение: показать условия отмены и политику обработки данных.

Элементы:
- условия отмены;
- правило 24 часов;
- заглушка возврата в MVP;
- политика обработки телефона, имён участников и данных об аллергиях;
- версия и дата обновления;
- кнопка назад.

Критерии приёмки:
- экран доступен из авторизации, карточки класса и подтверждения;
- текст понятен клиенту;
- данные приходят из API или согласованной статической версии.
