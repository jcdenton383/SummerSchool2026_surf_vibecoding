## Описание

После выхода из аккаунта в расписании класс, на который клиент был записан, продолжал отображаться как `Вы записаны на этот класс`.

## Шаги воспроизведения

1. Авторизоваться.
2. Открыть SCR-003.
3. Убедиться, что забронированный класс помечен как записанный.
4. Перейти в SCR-010.
5. Нажать `Выйти`.
6. Вернуться или попасть в расписание.

## Фактический результат

Расписание продолжает использовать старые данные `bookings` и показывает персональную пометку для незалогиненного пользователя.

## Ожидаемый результат

После logout:

- пользователь считается неавторизованным;
- персональные пометки броней в расписании исчезают;
- кэш моих записей и уведомлений не используется для UI незалогиненного пользователя.

## Причина

React Query сохранял старое `bookingsQuery.data`, даже когда запрос становился `enabled: false` после logout. Экран расписания строил `activeBookingsBySlot` по старому кэшу без проверки наличия токена.

## Исправление

- В `frontend/app/schedule/index.tsx` добавлена проверка `if (!token) return empty map`.
- В `frontend/app/account/index.tsx` logout очищает персональные query-кэши и возвращает пользователя на `/schedule`.

## Изменённые артефакты

- `frontend/app/schedule/index.tsx` — персональные метки строятся только при наличии токена.
- `frontend/app/account/index.tsx` — logout очищает персональный React Query cache.
- `logs/BUG-20260706-stale-booking-marker-after-logout.md` — создан баг-лог.
